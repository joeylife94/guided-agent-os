# Source Citation Policy

## Citation Requirements
Every recommendation, answer, or piece of advice must include proper source citations.

## Citation Format
Citations must include:
- **Source Type**: Domain Knowledge, Policy, Tool Catalog, or External Reference
- **Document Title**: Name of the source document
- **Section**: Relevant section or heading within the document
- **Relevance Score**: Confidence that this source applies to the query (High/Medium/Low)

## Citation Examples
```
[Policy] Agent Behavior Policy - "Decision-Making Standards"
[Domain] Internal Operation Manual - "Workflow Stages"
[Tool] Approved Tools - "Legacy Database Access Configuration"
```

## Multi-Source Scenarios
When multiple sources inform a single recommendation:
- List all applicable sources
- Indicate if sources agree or conflict
- If conflicting, escalate to human review

## No Citation Cases
If no reliable source can be found to support a response:
- Explicitly state "No matching knowledge base entry"
- Request human guidance
- Do not fabricate sources
- Document this gap for future knowledge base enhancement
